## ❌ What Went Wrong

1. **Duplicate Ticket Explosion (Critical)**  
   The v3 Orchestrator's `trigger_sprint_2_planning()` function ran in an infinite loop, creating **226 duplicate "Dockerize" tickets** in `qorvari-docs`. Root cause: no deduplication check before issue creation, and the trigger fired every 4 heartbeat cycles indefinitely.  
   **Resolution:** Function removed entirely in Orchestrator v4. Ticket creation is now CEO-driven via the Dashboard UI.

2. **Orchestrator Skipping Planned Tickets**  
   The v3 Orchestrator had `sprint-planning` in its label skip-list, which meant it ignored every ticket that had been formally planned. Tickets sat in "To Do" indefinitely.  
   **Resolution:** v4 Orchestrator picks up ANY ticket without `agent-executing`, `qa-review`, or `ceo-approval` labels.

3. **Sprint Selector Allowed Forward Navigation**  
   The CEO could click ▶ past the active sprint into empty future sprints, causing confusion.  
   **Resolution:** Forward arrow removed. Only ◀ for history review + "Finalize Sprint" to advance.
