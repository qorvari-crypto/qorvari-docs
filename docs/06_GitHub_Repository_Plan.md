# Qorvari GitHub Repository Plan

## Module Audit: Duplicate & Overlap Analysis

Before creating a single repo, I audited every directory in the workspace. Here are the findings:

### DUPLICATES & OVERLAPS FOUND

| Local Directory | Duplicate Of | Evidence | Verdict |
|:---|:---|:---|:---|
| `serviceDiscovery/Initiator` | `ServiceDiscoveryCursor` | Same `pom.xml` structure, same Dockerfiles, same `errormessage.properties`. `ServiceDiscoveryCursor` is a Cursor AI-refactored copy. | **MERGE** — Use `serviceDiscovery/Initiator` as canon. |
| `serviceDiscovery/repos` | `serviceDiscovery/matilda-sd-executor` | Both contain `servicediscovery-executor.iml`, identical `pom.xml` and Ansible playbooks. `repos` is an older snapshot. | **MERGE** — Use `matilda-sd-executor` as canon. |
| `serviceDiscovery-MCCDDEV-backup-05272025` | N/A (Database dump) | Contains raw `.bson` MongoDB backup files (~2.8 GB). Not code. | **EXCLUDE** — Archive only. Never push to GitHub. |
| `Cusor Generated Docs` | N/A (AI-generated docs) | Contains generated Markdown docs from Cursor sessions. | **EXCLUDE** — Will be regenerated by our Tech Writer Agent. |
| `Status Reports` | N/A (PowerPoint slides) | Weekly status `.pptx` files. Corporate artifacts. | **EXCLUDE** — Not code. |
| `discoveryinitiator` | `discoveryexecutor` | Both are Python Flask microservices with identical project scaffolding (`setup.py`, `build-wheel.py`, `matilda_discovery/` package). They are **different services** but share a common library structure. | **SEPARATE REPOS** — Distinct services, but will share a common Python library repo. |
| `Kotlin` | N/A (Experimental) | Small Spring Boot Kotlin project. No overlap with main modules. | **SEPARATE REPO** — May be a modernization prototype. |

## The Qorvari GitHub Repo Plan

We will create **13 repositories** under the `qorvari` GitHub Organization. These are grouped by business domain to avoid fragmentation while keeping microservice boundaries clean.

### Tier 1: Open Source (Public Repos)

| # | Repo Name | Source Modules | Description |
|:--|:----------|:---------------|:------------|
| 1 | `qorvari-discovery-initiator` | `discoveryinitiator` | Python service: triggers infrastructure scans |
| 2 | `qorvari-discovery-executor` | `discoveryexecutor` | Python service: executes network/cloud scans |
| 3 | `qorvari-discovery-processor` | `discoveryresponseprocessor` | Python service: transforms raw scan data |
| 4 | `qorvari-discovery-core` | `serviceDiscovery/Initiator` + `serviceDiscovery/matilda-sd-executor` | Java monorepo: the core discovery backend (Initiator + Executor combined) |
| 5 | `qorvari-assessment-engine` | `assessmentengine` | Python service: cloud readiness scoring |
| 6 | `qorvari-ui` | `mtl-ui/matilda-ui` | Angular NX monorepo: the primary web dashboard |

### Tier 2: Premium / Enterprise (Private Repos)

| # | Repo Name | Source Modules | Description |
|:--|:----------|:---------------|:------------|
| 7 | `qorvari-geniac` | `cloud-geniac-manage` | Python/Terraform: IaC template generation |
| 8 | `qorvari-iasc` | `iasc` | Python: migration tooling and IaC security |
| 9 | `qorvari-optimize` | `optimize` | Java: cost optimization and rightsizing engine |
| 10 | `qorvari-aiops` | `aiops-ui-svc` | Java: AIOps anomaly detection backend |

### Tier 3: Shared Services (Private Repos)

| # | Repo Name | Source Modules | Description |
|:--|:----------|:---------------|:------------|
| 11 | `qorvari-services` | `dynamic_dashboard` + `email-plugin` + `lookup` + `AssetProcessor` | Java monorepo: shared utility microservices bundled together to reduce repo sprawl |
| 12 | `qorvari-platform` | *(New)* | The Orchestrator, Agent Registry, Agent Comms, CI/CD workflows, and `docker-compose.yml` |
| 13 | `qorvari-docs` | *(New)* | ADRs, PRDs, Runbooks, API specs, and all industrial-standard documentation |
